# 수동 QA 시나리오 체크리스트

배포 전 아래 시나리오를 수동으로 점검한다. `npm run dev` 실행 후 `npm run db:seed`로 생성된 계정
(교사: `teacher@example.com`, 학생: `student1~4@example.com`, 학부모: `parent1@example.com`,
`parent2@example.com`, 공통 비밀번호 `Passw0rd!`)을 사용하거나 직접 회원가입하여 진행한다.

## 1. 인증 / RBAC

- [ ] 학생/학부모/교사 각 역할로 회원가입이 가능하다.
- [ ] 학부모 회원가입 시 자녀 이메일을 2명 이상 등록할 수 있다.
- [ ] 존재하지 않는 이메일로 로그인하면 적절한 오류 메시지가 뜬다.
- [ ] 잘못된 비밀번호로 로그인하면 적절한 오류 메시지가 뜬다.
- [ ] 로그인하지 않은 상태로 `/dashboard`, `/teacher`, `/results` 접근 시 `/login`으로 리다이렉트된다.
- [ ] 로그인 후 `/login`, `/signup` 접근 시 `/dashboard`로 리다이렉트된다.
- [ ] 로그아웃 후 다시 보호된 페이지 접근 시 로그인 화면으로 이동한다.
- [ ] 학생 계정으로 다른 학생의 제출물(API `/api/submissions/[id]`)에 접근 시 403이 반환된다.
- [ ] 담당하지 않는 학급/평가에 교사가 접근 시 403이 반환된다(자동화 테스트로 검증됨: `npm test`).

## 2. 교사 — 수행평가 등록 및 채점

- [ ] 학급을 생성할 수 있다.
- [ ] 학생 이메일로 학급에 학생을 등록할 수 있다.
- [ ] 수행평가 항목(제목/교과/마감일/루브릭)을 등록할 수 있다.
- [ ] 등록 즉시 학급의 모든 학생에 대해 제출 현황(미제출) 행이 생성된다.
- [ ] 점수/피드백을 입력하고 "임시저장" 시 공개 여부는 변하지 않는다("채점 중" 유지).
- [ ] 개별 학생에 대해 "공개" 클릭 시 해당 학생만 결과가 공개된다.
- [ ] "학급 전체 일괄 공개" 클릭 시 채점된 모든 학생이 한번에 공개된다.
- [ ] 점수 없이 개별/일괄 공개를 시도하면 실패하거나 해당 학생은 건너뛴다.

## 3. 학생 / 학부모 결과 확인

- [ ] 학생 계정으로 로그인 시 본인의 평가 목록(교과별)이 보인다.
- [ ] 공개되지 않은 평가는 점수/피드백 없이 "채점 중"으로만 표시된다.
- [ ] 공개된 평가는 점수, 루브릭, 피드백이 모두 표시된다.
- [ ] 학부모 계정으로 로그인 시 화면 상단에 자녀 선택 드롭다운이 보인다(자녀 2명 이상인 계정으로 확인).
- [ ] 자녀를 전환하면 해당 자녀의 결과만 표시된다.
- [ ] 자녀가 아닌 학생 ID를 URL에 직접 넣어도 접근할 수 없다(다른 자녀 정보로 안내되거나 접근 거부).
- [ ] 학기별/교과별 점수 추이 차트가 공개된 점수만으로 표시된다.
- [ ] 모바일 폭(약 375~430px)에서 목록/상세/차트 화면이 가로 스크롤 없이 보인다.

## 4. 알림

- [ ] 교사가 결과를 공개하면 학생과 연결된 학부모 모두에게 알림 메일이 발송(또는 SMTP 미설정 시 로그)된다.
- [ ] 동일 결과를 다시 공개해도 같은 사람에게 중복 메일이 가지 않는다.
- [ ] `/api/notifications/run-reminders`를 올바른 `x-cron-secret` 없이 호출하면 401이 반환된다.
- [ ] 마감 24시간 이내 미제출 평가에 대해 학생/학부모에게 "마감임박" 메일이 발송된다.
- [ ] 마감이 지난 미제출 평가에 대해 학생/학부모에게 "미제출" 메일이 발송된다.
- [ ] 동일 스윕을 재실행해도 중복 발송되지 않는다.

## 5. 배포 / CI

- [ ] `npm run build`가 오류 없이 성공한다.
- [ ] `npm run lint`가 통과한다.
- [ ] `npm test`가 통과한다(RBAC, 공개 전/후 노출, 알림 중복방지 포함).
- [ ] CI 워크플로우(`.github/workflows/ci.yml`)가 실제 PostgreSQL 서비스 컨테이너에서 `migrate deploy` 후
      통과한다.
- [ ] `.env.example`에 정의된 모든 환경변수가 배포 환경에 설정되어 있다.
